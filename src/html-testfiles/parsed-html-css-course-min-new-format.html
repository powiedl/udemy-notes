42. The Switch Statement

switch (variable) {
  case 1:
    console.log('variable hat den Wert 1');
    break;
  case 2:
    console.log('variable hat den Wert 2);
    break;
  case 3:
  case 4: // eher unüblich, aber möglich

Abschnitt 5: Capture Collections of Data with Arrays

47. Push and Pop

Array Methoden - um Elemente hinzuzufügen oder zu entfernen:

push ... fügt am Ende hinzu - Rückgabewert: Neue Länge vom Array

pop ... entfernt vom Ende - Rückgabewert: das entfernte Element

shift ... entfernt vom Anfang - Rückgabewert: Neue Länge vom Array

unshift ... fügt am Anfang hinzu - Rückgabewert: das entfernte Element


ACHTUNG: Man kann auch mehrere Elemente auf einmal mit unshift hinzufügen. Wenn man das macht, werden die Elemente in der angegebenen Reihenfolge am Anfang eingefügt. Wenn man die Elemente einzeln (in der gleichen Reihenfolge) unshiftet, wird die Reihenfolge "umgedreht" bzw. wird zuerst das erste Element hinzugefügt (und das ergibt dann das neue Array). Zu diesem neuen Array wird dann das nächste Element am Anfang hinzugefügt - und damit ist die Reihenfolge im finalen Array dieser beiden Elemente umgekehrt wie der Aufruf war.

95. Hoisting

hoisting ... Eine Variable kann benutzt werden, bevor sie deklariert wurde (das trifft aber nur auf var zu). Bei let und const bekommt man in so einem Fall den ReferenceError.


Sie hat dann den Wert undefined. Wenn die Variable im gesamten Block, wo sie verwendet wird, nicht deklariert wird, bekommt man - beim Versuch auf die Variable zuzugreifen - einen ReferenceError. Wenn sie nach dem Zugriff (im gleichen Block) deklariert wird, bekommt man den Wert undefined. Beim hoisting wird also nur die Deklaration "nach vorne" gezogen, die Wertzuweisung bleibt an der Stelle wo man im Code die Variable (mit Wertzuweisung) definiert.


Bei Funktionsdeklarationen passiert das ebenfalls, d. h. man kann eine Funktion aufrufen, bevor sie deklariert wurde. Das Beispiel funktioniert (und ist valide)


howl();
function howl() {
   console.log('AWOOOO');
}


Function Expressions verhalten sich so wie Variablen (bei let und const). Können also nicht davor verwendet werden.

101. Array.find

Für die folgenden Array-Methode ist der erste Parameter immer eine Callback-Funktion, die true oder false zurückgeben muss.

arr.find ... Wenn true zurückgegeben wird, gilt das Element als "gefunden" und wird selbst von arr.find zurückgegeben. Danach hört die Verarbeitung auf - es wird als nur das ERSTE Element, dass der callback-Funktion "genügt" zurückgegeben.

arr.some ... Wenn die Callbackfunktion für ZUMINDEST EIN Elemente von arr true zurückgibt, gibt every auch true zurück, sonst false.

104. Revisiting Sort!

arr.sort ... Standardmäßig werden alle Elemente des Arrays in eine Zeichenkette konveritiert und dann werden die Zeichenketten alphabetisch verglichen. Oft will man etwas anderes (z. b. Zahlen nach ihrem Wert sortieren). In dem Fall kann man sort eine callback-Funktion übergeben. Diese muss zwei Parameter akzeptieren (die zu vergleichenden Elemente). Wenn das Ergebnis <0 ist, kommt das erste Element vor dem zweiten. Wenn es 0 ist, behalten beide Elemente ihre Position zueinander. Wenn es größer als 0 ist, kommt das zweite Element vor dem ersten.


Um Zahlen aufsteigend zu sortieren, kann man einfach diese Funktion verwenden:

arr.sort((a,b) => a-b);

Um Zahlen absteigend zu sortieren:

arr.sort((a,b) => b-a);


ACHTUNG: sort verändert das eigentliche Array! Es wird keine "Kopie" zurückgegeben.

105. Reduce Intro

arr.reduce ... Man reduziert ein Array solange, bis ein Wert übrig bleibt. Das ist im Normalfall kein Element an sich, sondern ein berechneter Wert. Die Callback-Funktion verlangt 2 Parameter. 1. accumulator (das wird am Ende das Ergebnis von reduce), 2. aktueller Wert. Beispielsweise kann man mit reduce die Summe aller Zahlen eines Arrays ermitteln:

arr.reduce((accumulator,currentValue) => { return accumulator + currentValue; })

Beim ersten Durchlauf wird das 1. Element vom Array als accumulator übergeben, das 2. Element als currentValue, bei jedem weiteren Durchlauf, wird der return Wert vom vorigen Durchlauf als accumulator übergeben und das jeweils nächste Element als currentValue.

Nach der callback-Funktion kann ein Startwert als weiterer Parameter übergeben werden, dann wird dieser Startwert als accumulator beim ersten Aufruf verwendet - und das erste Element ist dabei dann der currentValue.

126. Annoyomatic Demo

Wenn in einem Objekt in einer Methode setInterval oder setTimeout aufgerufen wird, ist der Ausführungskontext für die aufgerufene Funktion global (sofern man nicht eine Arrow function benutzt, da dort ja der Ausführungskontext nicht geändert wird) - und nicht das Objekt. Damit ist this auch nicht mehr das Objekt - und man kann so nicht mittels this auf Attribute oder Methoden des Objekts zugreifen.

Daher lässt sich das Problem leicht umgehen, indem eben eine Arrow function verwendet wird. Statt

const o = {
  p:[1,2,3];
  m() {
    setInterval(function() {
      console.log(this.p[1]); // geht nicht, weil sich der Ausführungskontext ändert
    },2000);
  }
}

also

...
    setInterval(() => {
      console.log(this.p[1]); // geht, weil die Arrow function den Ausführungskontext nicht ändert
    }
...

Darum verwendet man im Normalfall für Methoden von Objekten die Kurzschreibweise und nicht die Arrowschreibweise, innerhalb von Methoden dann aber Arrow functions als anoynyme.

Abschnitt 16: Asynchronous Code, Callbacks & Promises

167. Introducing Promises!

Ein Promise (object) ist die eventuell erfolgreiche (oder nicht erfolgreiche - failure) Ausführung von einem Code. Ein Promise hat immer zwei Callback-Parameter - einen für die erfolgreiche und einen für die failure Ausführung. Der erfolgreiche wird ausgeführt, wenn die Ausführung erfolgreich war, der failure, wenn es nicht erfolgreich war.

Der prinzipielle Aufbau - wobei dieses Versprechen nie "aufgelöst" werden kann, weil weder resolve noch reject aufgerufen wird. Das Versprechen ist "ewig" im Status "pending":

const meinVersprechen = new Promise((resolve, reject) => {
})


Ein Beispiel, wo das Versprechen aufgelöst wird (setTimeout wird nur verwendet um das Ergebnis zu verzögern):

const meinVersprechen = new Promise((resolve, reject) => {
    setTimeout(() => {
        const r=Math.random();
        if (r<.5) {
            resolve();
        } else {
            reject();
        }
    },5000);
})



5:16

Man kann dann meinVersprechen aufrufen. Solange weder der resolve() noch der reject() Aufruf erreicht wurde, hat das meinVersprechen den PromiseState 'pending', wenn resolve() aufgerufen wird, hat es den PromiseState 'fulfilled' und wenn reject() aufgerufen wird, hat es den PromiseState 'rejected'.

Aber - PromiseState ist kein Attribut vom Promise, d. h. ich weiß (im Moment nicht) wie man es programmtechnisch abfragt. In der Console kann man einfach meinVersprechen aufrufen, dann sieht man unter "Promise" den PromiseState. Aber Promise ist kein Attribut von meinVersprechen (genausowenig wie PromiseState).


Ein Versprechen hat aber zwei Attribute, die Funktionen sind (wobei nicht unbedingt beide angegeben werden müssen):

.then - wird ausgeführt, wenn im Promise resolve() ausgeführt wird.

.catch - wird ausgeführt, wenn im Promise reject() ausgeführt wird.

meinVersprechen.then(() => { console.log('JA') }).catch(() => { console.log('NO'); })

Abschnitt 18: Async & Await: JS Magic

182. The Async Keyword

wenn man vor eine Funktion async schreibt, gibt sie ein Promise zurück (auch wenn man das selbst so nie schreibt).

async function f() {
  return 'ERGEBNIS';
}

der Aufruf f() liefert dann

Promise { <fulfilled>: 'ERGEBNIS' }

Und auch, wenn man das Ergebnis in einer Variablen speichert - es bleibt ein Promise ...

const s = f();

Um das Ergebnis weiterzuverarbeiten, muss man also darauf warten (.then):

f().then(erg => { console.log(erg) })

liefert das erwartete 'ERGEBNIS' als Konsolenausgabe (wobei trotzdem noch das Promise retour kommt.

Um in einer async function das zurückgelieferte Promise zu rejecten muss man einen Errow thrown:

async function add(x,y) {
  if (typeof x === 'number' && typeof y === 'number') {
    return x+y;
  } else {
    throw 'both parameters must be numbers';
  }
}

Und so kann es dann aufgerufen werden:

add('asdf',5).then(val => console.log(val)).catch(err => console.log('ERROR:',err))

Dem Aufruf kann/sollte man mit await einleiten, dann wird die ganze Promise"geschichte" wegabstrahiert. Es wirkt so als ob die aufgerufene Funktion gar kein Promise erstellt.

const s=await add('a',5).catch(e => console.log(e));

191. Constructor Functions

Constructor für eine Klasse. Eigentlich eine normale Funktion (Konvention: 1. Buchstabe in Großbuchstaben). In dieser Funktion kann man this verwenden. Color.prototype.rgb = function() {
  const {r,g,b} = this;
  return `rgb(${r}, ${g}, ${b})`;
}
