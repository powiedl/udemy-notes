Abschnitt 10: Selectors Pt. 1

75. The Attribute Selector

Für attributebased Selektoren gibt es neben = (das Attribut muss exakt dem Wert auf der rechten Seite des = entsprechen) noch andere Vergleichsoperatoren:

  • *=    der Wert rechts muss irgendwo im Attribut vorkommen

  • ^=    das Attribut muss mit dem Wert rechts anfangen

  • $=    das Attribut muss mit dem Wert rechts aufhören

Wenn man ein Attribut hat, dass mehrere Werte annehmen kann (z. b. class) und man will Elemente selektieren, die eine Klasse haben, die mit einem bestimmten String beginnen muss man *= und ^= kombinieren.

Bsp: man sucht divs die eine Klasse haben, die mit "lecture-*" beginnt:

div[class^="lecture-",class*=" lecture-"]

Dabei ist das Leerzeichen vor dem zweiten lecture- wichtig, damit immer der Anfang der Klassennamen verglichen wird (ein div mit class="bla bla-lecture-" wird somit nicht gefunden - ohne führendes Leerzeichen würde es gefunden werden - aber dann bräuchte man auch den ersten Teil nicht, dann wäre es einfach Teilstring).

margin collapsed vertikal, d. h. wenn zwei Elemente aufeinandertreffen mit jeweils 20px vertikalem margin ist zwischen den beiden nur 20px vertikaler Abstand, nicht 40px. Wenn eines 20px und das andere 30px hat, beträgt der Abstand 30px (also der größere Abstand).

114. Wrapping Up The Cascade

Das Cascading in CSS (welche CSS Regel gilt - ein Teil davon ist die Specificity):

  1. Importance (im wesentlichen "!important")

  2. Origin

  3. Specificity

  4. Position


Für Origin gilt dabei:

Legt fest, wo die CSS Regel festgelegt wurde. Das kann (in absteigender Wichtigkeit - für die Auswahl welche Regel zur Anwendung kommt) sein:

  1. in der Webseite (wobei da genauso die dort verlinkten CSS-Files dazuzählen)

  2. Vom User festgelegt (wie auch immer das geht)

  3. Browser defaults


Wenn Importance, Origin und Specificity für einen Konflikt genau das gleiche Ergebnis haben, d. h. es konnte noch keine Entscheidung getroffen werden, welche Regel zur Anwendung kommt, entscheidet die Position, wo die Regel definiert wurde. Die letzte Position gewinnt, darum sollte man beispielsweise CSS Frameworks möglichst weit oben im Dokument verlinken, dann kann man einzelne Einstellungen davon später im eigenen Stylesheet überschreiben.

Abschnitt 15: CSS Units

125. The Joy of Rem Units

rem (root em) beziehen sich auf die eingestellte Schriftgröße im Browser, d. h. wenn ein User eine sehr große Schrift auswählt und ein anderer eine sehr kleine wird eine Webseite, die die Schriftgröße auf 1rem eingestellt hat bei beiden Usern sehr unterschiedlich aussehen - aber für den jeweiligen User immer "passend". Wenn man fix Pixel verwendet, wird die Einstellung vom Browser vom User "ignoriert" - das sollte man eher vermeiden, weil der User wird sich was bei der Wahl der Schrifteinstellung gedacht haben ...

126. Understanding The Em Unit

em sind manchmal relativ zum jeweiligen Parent Element, wohingegen rem zum root Element relativ sind. Wenn man jetzt folgenden Code hat, wird die Schrift sehr schnell sehr viel größer:


HTML:

<ul>
  <li> Erster Oberpunkt
    <ol>
       <li> Erster Unterpunkt</li>
       <li>Zweiter Unterpunkt</li>
    </ol>
  </li>
  <li> Zweiter Oberpunkt</li>
</ul>

CSS:

ul, ol, li {
  font-size: 2em;
}


em sind manchmal aber auch relativ zur font-size des aktuellen Elements (z. b. bei margin). Besonders "komisch" wird es, wenn die font-size des aktuellen Elements auch in em angegeben ist. Dann wird im ersten Schritt die font-size ausgerechnet (wieviele Pixel sind es wirklich - abhängig von der font-size im übergeordneten Element) und anschließend wird das margin anhand der jetzt berechneteten font-size des eigenen Elements berechnet).

128. Which Units Should You Use?

Rule of thumb:
Wann verwendet man welche CSS Units:

  • px - eher nicht für Schriftgrößen, aber gut für kleine Details (z. b. border, shadow).

  • rem/em - gut für Schriftgrößen und oft auch für padding/margin (weil das dann auch relativ zur Schriftgröße ist)

  • % - gut für das Festlegen von Layouts und Breiten

  • vh/vm - gut für "globale" Layouteinstellungen (100vh [die ganze sichtbare Höhe] oder 50vw (die halbe sichtbare Breite])

Abschnitt 18: Positioning

150. Relative Positioning

position

  • static - default. Das Element ist normal im Document flow. Die Attribute top, left, bottom, right und z-index haben keine Auswirkung.

  • relative: Das Element ist normal im Document flow, aber mittels top, left, bottom, right und z-index kann die Anzeige für das Element verschoben werden.

  • absolute: Das Element verschwindet aus dem normalen Document flow (sein Platz wird also nicht länger "reserviert"). Ohne weitere Angabe von top, left, ... bleibt es aber an der gleichen Stelle - nur die nachgelagerten Elemente "rutschen" nach. Wenn man dann top, left, ... festlegt, bezieht sich das auf das gesamte Dokument, d. h. top=0;left=0; führt dazu, dass das Element ganz nach links oben vom Dokument verschoben wird.
    Wenn das/ein Parent Element selbst position <> static hat, bezieht sich die Positionsangabe des Elements auf das "erste" Parent Element mit position.


scheinbar kann man aber top und bottom bzw. left und right nicht gleichzeitig festlegen.

  • fixed: Wie absolute, aber es bezieht sich immer auf das root Element (egal, ob es am Weg dazwischen irgendwelche Elemente gibt, die selbst positioned sind).

Abschnitt 20: Flexbox: Part 1

171. Working With Flex-Direction

flex-direction legt fest, ob die Main Axis horizontal oder vertikal verläuft:

  • row: main-axis verläuft horizontal (von links nach rechts) (Standardwert) - hängt aber auch von der Einstellung der Schriftrichtung ab (wenn die Schriftrichtung von rechts nach links ist, z. b. Arabisch, verläuft row horizontal - aber von rechts nach links)

  • column: main-axis verläuft vertikal (von oben nach unten)

  • row-reverse: main-axis verläuft horizontal (von rechts nach links)

  • column-reverse: main-axis verläuft vertikal (von unten nach oben)


0:25

173. Flex-Wrap

flex-wrap legt fest, ob Elemente in eine neue Zeile (bei flex-direction row) oder eine neue Spalte (bei flex-direction column) wechseln, wenn sie nicht genug Platz haben.

  • nowrap (Standard): keine neue Zeile/Spalte (statt dessen werden die Elemente solange verkleinert, bis sie alle in eine Zeile/Spalte passen)

  • wrap: Elemente kommen in eine neue Zeile/Spalte, wenn sie nicht mehr in eine Zeile/Spalte passen- hier werden sie auch nicht verkleinert.

Dieses Verhalten gilt aber nur, wenn der Container in der entsprechenden Ausdehnung eine Größe festgelegt hat (bei row also width, bei column height). Wenn das nicht der Fall ist, wächst der betreffende Container - wenn er kann (z.b. wenn es max-width oder max-height nicht verhindert), damit alle Elemente hineinpassen (ohne wrap - egal wie die flex-wrap Einstellung ist)

1:23

Fortsetzung flex-wrap:

  • wrap-reverse: Elemente kommen in eine neue Zeile/Spalte, wenn sie nicht mehr in eine Zeile/Spalte passen, aber die neue Zeile/Spalte entsteht entgegen der Richtung der zweiten Achse. Bei flex-direction row wird die neue Zeile oberhalb der bisherigen Zeile gemacht (bei wrap unterhalb).
    Bei flex-direction column wird die neue Spalte links der bisherigen Spalte gemacht (bei wrap rechts davon) - jeweils bei Schriftrichtung links nach rechts.

4:15

174. Justify-Content

justify-content: bestimmt wie die Elemente entlang der Hauptachse verteilt werden. Es gibt folgende Einstellungen:

  • flex-start (Standard): Die Elemente werden bündig am Anfang der Hauptachse verteilt.

  • center: Die Elemente werden bündig zentriert angeordnet.

  • flex-end: Die Elemente werden bündig am Anfang der Hauptachse verteilt.

  • space-between: Von Anfang an bis zum Ende werden die Elemente verteilt. Der "freie Platz" wird gleichmäßig dazwischen verteilt.

  • space-around: Hier wird auch von Anfang bis zum Ende verteilt, aber es wird mit freiem Platz begonnen und aufgehört.

0:15

176. Align-Items

align-items: Dient dazu die Elemente entlang der Nebenachse (cross axis) auszurichten. Es gibt folgende Einstellungen:

  • stretch (Standard): Jedes Element füllt den ganzen Bereich der Nebenachse (also vom Anfang bis zum Ende der Nebenachse) - aber nur, solange bei den Elementen height (bei flex-direction row) bzw. width (bei flex-direction column) nicht gesetzt ist - ansonsten wird die Einstellung verwendet (und das Element startet am Anfang der cross axis)

  • flex-start: Die Elemente beginnen am Anfang der cross-axis.

  • center: Die Elemente werden in der cross-axis zentriert.

  • flex-end: Die Elemente hören am Ende der cross-axis auf.


0:33

178. Align-Content

align-content: passiert nur, wenn es mehrere rows oder columns gibt, d. h. nur wenn flex-wrap nicht auf nowrap gesetzt ist. Folgende Werte sind möglich:

  • flex-start: die einzelnen rows bzw. columns werden am Anfang gestapelt (der freie Bereich zwischen ihnen befindet sich nur am Ende).

  • center: die einzelnen rows bzw. columns werden in der Mitte gestapelt (freier Bereich gleichmäßig davor und danach).

  • flex-end: die einzelnen rows bzw. columns werden am Ende gestapelt (freier Bereich nur am Anfang).

  • space-around: freier Bereich wird gleichmäßig zwischen (und auch vor der ersten nach der letzten) verteilt.

  • space-between: freier Bereich wird gleichmäßig zwischen ihnen verteilt. Die erste fängt ganz am Anfang an und die letzte endet ganz am Ende.

0:17

180. Align-Self

align-self - im Prinzip das gleiche wie align-items, ABER es wird einem einzelnen Element zugewiesen. Damit kann man die Ausrichtung eines einzelnen Elements an der cross-axis anders wie alle anderen festlegen. Die möglichen Werte sind aber dieselben:

  • stretch

  • flex-start

  • center

  • flex-end

Abschnitt 21: Flexbox Pt 2: Flex Items & Patterns!

182. The Flex-Grow Property

flex-grow: wird auch auf den Elementen gesetzt und bestimmt wieviel extra Platz sich das betreffende Element nimmt. Es geht dabei um das Verhältnis von flex-grow zwischen den verschiedenen Elementen (die flex-grow gesetzt haben).


Wenn ein Element flex-grow auf 1 hat und ein anderes flex-grow auf 2, wächst das mit 2 doppelt so schnell wie das mit 1. max-width/max-height wird dabei von flex-grow berücksichtigt, d. h. wenn das Element an seinem Limit angekommen ist, wächst es nicht mehr weiter.


flex-shrink: Analaog zu flex-grow, aber es geht darum, wieviel Platz ein Element "verliert" wenn insgesamt nicht genug zur Verfügung steht. ACHTUNG: flex-shrink hat einen Default von 1! Wenn man also nicht will, dass ein Element shrinkt, muss man flex-shrink auf 0 setzen.

10:58

184. Flex-Basis: Important & Confusing

flex-basis: Legt die Anfangsgröße des Elements fest. Im Prinzip macht es genau das gleiche wie width (bei flex-direction: row) oder height (bei flex-direction: column). Der Vorteil von flex-basis ist, dass es gemeinsam mit flex-grow und flex-shrink in der Abkürzung flex zusammengefasst werden kann.

flex: flex-grow flex-shrink flex-basis;

flex hat zusätzlich noch ein paar Möglichkeiten:

  • initial: übernimmt width bzw. height als flex-basis, 0 für flex-grow und 1 für flex-shrink (d. h. das Element ist max. width breit)

  • auto: wie initial, aber auch flex-grow ist 1

  • none: wie initial, aber auch flex-shrink ist 0 (d. h. die Größe ist unveränderlich)

2:04

185. The Flex Shorthand Property

In den Chrome Development Tools kann man unter Layout ganz unten Flexbox overlays einschalten und in der Elements Ansicht im HTML Sourcecode ein Element auswählen (und dann sieht man das flex Attribut auf der Webseite entsprechend dargestellt, mit wie groß ist flex-basis und was ist der extra Platz bzw. der "verlorene" Platz).

5:51

186. The Order Flex-Item Property

order: Verändert die Reihenfolge, wie Elemente gerendert werden (im Normalfall bestimmt das HTML die "Reihenfolge"). order ist eine Zahl, je höher die Zahl, desto weiter ans Ende rutscht das betreffende Element. Elemente mit gleicher order werden wie im HTML festgelegt gerendert, d. h. wenn zwei Elemente die order 3 haben, kommt zuerst das erste Element (im HTML) mit order 3 und daran anschließend das zweite mit order 3.

Abschnitt 23: Making Responsive Websites With Media Queries

202. Our First Media Query

Media Queries: Starten immer mit @media, dann kommen die Bedingungen, unter denen die Mediaquery gilt, dann kommen die Styles, z. b. 


body {
    background-color: red;
}
@media (min-width: 800px) {
    body {
        background-color: blue
    }
}

In diesem Beispiel ist der Body rot - aber ab 800 px Breite ist er blau; Man kann auch mehrere Media Queries haben, z. b. eine weitere mit 

@media (min-width: 1200px) {
    body {
        background-color: green;
    }
}

Dann ist der Body bis 800px rot, dann bis 1200px blau und darüber dann immer grün. Die Styles gelten also immer dann, wenn die Bedingung erfüllt ist.


Und ein Media Query bezieht sich auf die Größe des Viewports (der Bereich des Bildschirms, der für den Inhalt der Webseite zur Verfügung steht) - nicht auf die Browserfenstergröße und auch nicht auf die Bildschirmgröße!


Media Queries müssen auch nach den "normalen" Styledefinitionen kommen, sonst werden sie von den normalen "überschrieben"!

6:02

203. More On Media Queries

@media only screen and ...


only ist dabei ein Schlüsselwort, dass verhindert, dass ein Browser, der Media-Queries nicht versteht die Styles in der Media Query immer anwendet.


screen definiert für welchen Ausgabetyp diese Query gilt, weitere Beispiele sind print (nur für einen Ausdruck) oder all (wenn es sowohl für den Bildschirm als auch den Ausdruck gilt).

Abschnitt 25: CSS Grid Part 1

218. Enabling CSS Grid

display: grid; ... Aktiviert das Grid-System am Grid-Container (ähnlich wie display: flex; das Flexsystem am Container aktiviert)

display: inline-grid; ... ähnlich wie display: grid; - aber der Grid-Container selbst verhält sich wie ein inline Element (für die Elemente innerhalb des Grid-Containers macht es keinen Unterschied).

grid-template-columns bzw. grid-template-rows ... bestimmt wie viele Spalten bzw. Zeilen das Grid hat, wobei man für jede Spalte/Zeile die Breite angibt. Es gibt fr als neue Einheit (fraction) - und steht für ein gleichmäßiges Verhältnis, d. h. grid-template-rows: 1fr 1fr; führt zu zwei gleich hohen Zeilen in dem Grid (grid-template-rows: 50% 50%; liefert genau das gleiche Ergebnis). 2fr 1fr führt ebenfalls zu 2 Zeilen, wobei die erste doppelt so hoch wie die zweite ist (2 Teile, 1 Teil).

Wenn man zuwenig Spalten definiert, wird eine neue Zeile begonnen. Wenn man zu wenig Zeilen definiert, quetscht sich der Rest in den verbleibenden Platz.

2:56

223. The Minmax() Function

minmax() für Spalten- oder Zeilengröße. Man gibt einen Bereich von - bis an, in dem sich die Größe befinden darf (macht dann Sinn, wenn der Gridcontainer selbst keine fixe Größe in der Dimension hat, sondern variabel ist). Wenn der erste Wert größer als der zweite ist, wird das ganze ignoriert.


Oft wird das so verwendet, dass ein fixes Minimum (px, rem, em) und ein flexibles Maximum (fr, %, ...) angegeben wird.


Als Maximum kann man auch auto angeben, dann wird die Dimension so gewählt, dass der gesamte Inhalt in den Zellen Platz hat.


repeat(number,value); kann verwendet werden, um sich Schreibarbeit zu ersparen repeat(3, 1fr); ist gleichbedeutend mit 1fr 1fr 1fr; (bzw. um im Fall des Falles weniger ausbessern zu müssen - wenn man auf einmal 2fr 2fr 2fr; verwendet will). Kann auch mit minmax() kombiniert werden, z. b. repeat(4,minmax(100px,1fr));

2:51

225. Fit-Content

fit-content(size) legt fest, dass die Größe ausreichend sein soll um den Inhalt aufzunehmen - aber maximal size groß, d. h. fit-content(300px) führt zu einer Größe von 0 (kein Content) bis 300px.

Mit column-gap und row-gap kann man bestimmen, wie viel Abstand zwischen den Spalten bzw. Zeilen sein soll. Der gap ist aber nur zwischen den Spalten bzw. Zeilen - nicht an der "Außenseite". Für die gap kann fr nicht verwendet werden (wird als "0" interpretiert); Kann auch als gap abgekürzt werden. Mit einem Wert wird das gleiche für row-gap und column-gap gesetzt, wenn zwei Werte angegeben sind, kommt (wie bei grid-template) zuerst die row und dann die column, also gap: 10px 30px setzt die row-gap auf 10px und column-gap auf 30px.

Abschnitt 26: CSS Grid Part 2

228. Working with Grid Lines

grid-lines können dazu verwendet werden, einzelne Elemente in einem Grid zu positionieren. Es gibt grid-lines für Spalten und welche für Zeilen. In den Dev Tools werden sie angezeigt, wenn man das Grid-Layout aktiviert. Und man kann damit auch sagen, dass ein Element mehr wie nur einen "Spot" besetzt.


grid-column-start, grid-column-end, grid-row-start und grid-row-end dient dazu, damit man ein einzelnes Grid Element positionieren kann. Man stellt dabei ein, bei welcher grid-line ein Element startet bzw. endet.


Wenn man eine Spaltenlinie/Zeilenlinie auswählt, die es eigentlich nicht gibt, wächst das Grid, damit es sie gibt.


Positive line-Nummern starten links bzw. oben, negative starten rechts bzw. unten. Für das Ende kann man auch span x angeben (x ist dabei die Anzahl an Spalten bzw. Zeilen, die das Element umfasst. -start: 2 -end:4 kann daher auch als -start:2 und -end: span 2 geschrieben werden).

4:23

231. The Grid-Area Shorthand Property

Für grid-column-start und -end (bzw, für row) gibt es folgende Abkürzung: grid-column: start / end; end kann dabei auch in Form von span angegeben werden.


Wenn für grid-column|-row nur eine Zahl angegeben wird, ist das der Anfang und das Element füllt in dieser Dimension einen Trackplatz. Wenn span angeben wird, wird das Element so weit wie möglich am Anfang positioniert und ist span x Elemente breit bzw. hoch.


grid-area ist (u. a.) eine noch kürzere Form: grid-area: a / b / c / d; a ... grid-row-start, b ... grid-column-start, c ... grid-row-end, d ... grid-column-end;


Es gibt für grid-area auch noch eine ganz andere Schreibweise.

3:45

233. Grid-Template-Areas

grid-template-area - damit kann man "Bereiche" benennen. Folgendes Beispiel: 

grid-template-areas: 
  "a a c"
  "b b c";
}


Damit definiert man 3 Bereiche. "a" umfasst die ersten beiden Spalten in der ersten Zeile, "b" umfasst die ersten beiden Spalten in der zweiten Zeile und "c" umfasst jeweils die letzte Spalte in beiden Zeilen. Jeder Bereich muss dabei ein Rechteck oder Quadrat bilden - folgendes ist daher nicht legal: "a b a".


Die Namen müssen nicht nur ein Zeichen lang sein. Durch die Leerzeichen wird der Name des nächsten Tracks festgelegt.


Und im Anschluss kann man Elemente positionieren, indem man ihnen einen Bereich zuweist:

grid-area: a;


Damit weist man dem Element den Bereich a zu, d. h. im Beispiel die ersten beiden Spalten in der ersten Zeile. WICHTIG: Bei grid-area darf man dann keine Anführungszeichen mehr verwenden.


1:01

234. Using Grid-Template-Areas With Rows & Cols

Man kann auch grid-template-rows und grid-template-columns und grid-template-areas gemeinsam verwenden. Dann definiert man zuerst die Dimension der einzelnen Elemente (mit grid-template-rows bzw. -columns) und danach weist man die jeweiligen Tracks benannten Bereiche zu, z. b.: 

.container {
  display: "grid";
  grid-template-rows: 1fr 3fr 1fr;
  grid-template-columns: repeat(4,1fr);
  grid-template-areas: 
    "a b b c"
    "a b b c"
    "d d d .";
}

.sidebar {
  grid-area: a;
}
.main {
  grid-area: b;
}
.aside {
  grid-area: c;
}
.footer {
  grid-area: d;
}

Wenn man beim grid-template-areas einzelne Tracks nicht verwenden will, kann man ihre Position mit einem . schreiben (weil die Anzahl der Elemente pro Zeile muss gleich sein, d. h. man kann nicht einfach was "weglassen").

0:15

235. Grid-Auto-Flow

grid-auto-flow bestimmt, wo neue Elemente dazu kommen, wenn es nicht mehr genug Tracks gibt. Der Standardwert ist row, d. h. in dem Fall wird eine neue Zeile gemacht. Man kann es aber auch auf column setzen, dann wir stattdessen eine neue Spalte begonnen, wenn mehr Elemente existieren wie Tracks vorhanden sind.


Außerdem bestimmt es auch, wie die Elemente gefüllt werden. Bei row wird eine Row "voll" gemacht, bevor in die nächste gewechselt wird. Bei column wird eine Spalte "voll" gemacht, bevor in die nächste gewechselt wird. MAn kann zusätzlich noch dense angeben (sowohl bei row als auch column). Dann wird versucht später hinzukommende Elemente in Lücken davor zu geben, wenn die Lücken groß genug sind.


grid-auto-rows und grid-auto-columns - damit kann man eine Höhe bzw. eine Breite für jene Elemente angeben, die sich im sauber definierten Grid nicht mehr ausgehen. Wenn man das nicht angibt, werden sie so klein wie möglich, dass ihr Inhalt passt, angelegt.

236. Grid Auto Columns & Rows

Bei repeat() kann man als Anzahl auch auto-fill und auto-fit angeben.


Im Normalfall hat der Grid-Container bei diesen Optionen eine definierte Breite bzw. Höhe (je nachdem ob, es um column oder row geht). Bei auto-fill werden so viele Elemente eingefügt, wie sich ausgehen, z. b. width: 700px; und grid-template-columns: repeat(auto-fill, 200px); - es werden 3 Spalten, weil drei Spalten 600px breit sind - vier aber 800px breit wären.


Bei auto-fit dürfen die Elemente weiter wachsen, wenn sich zusätzliche Elemente ausgehen würden (womit kein Platz ungenutzt bleibt). Bei auto-fill wachsen die Elemente nicht - wenn man vier Elemente hat, aber Platz für fünf wäre, bleibt der Platz bei auto-fill frei, bei auto-fit werden die vier Elemente um jeweils 25% größer.


Man kann auto-fill und auto-fit kann man für die Größe auch noch minmax() verwenden, damit "kein" ungenutzter Rand am Ende entsteht (Ausnahme: voriger Absatz).

Abschnitt 28: Building Animations

250. More Practice With @Keyframes 

@keyframes name {
  from {
    property: value;
  }
  to {
    property: differentvalue;
  }
}

Definiert eine Animation mit dem Namen name. Diese kann dann auf ein oder mehrere Elemente angewandt werden. 


el {
  animation-name: name;
  animation-duration: 3s;
}


Damit wird die Animation auf das oder die Elemente el angewendet. Die obige Definition ist die einfachste - mit einem Start- und einem Endframe. Man kann beliebig viele Keyframes definieren. Es müssen auch nicht in allen Keyframes alle Properties einen neuen Wert erhalten. Wenn man mehrere Keyframes definieren will, ist die Syntax so: 


@keyframes name {
  0% {
    property1: value1a;
    property2: value2a;
  }
  30% {
    property1: value 1b;
  }
  100% {
    property1: value1c;
    property2: value2b;
  }
}


Damit definiert man drei Keyframes - bei 0%, 30% und 100%. Beim zweiten Keyframe wird aber nur für property1 ein neuer Wert gesetzt, damit macht property2 eine Animation durch (Wie from und to).

256. The Animation Fill Mode Property

animation-fill-mode - damit kann man bestimmen, was am Ende der Animation passiert. Folgende Werte sind möglich:

  • none (Standard) ... Am Ende der Animation werden alle Änderungen sofort rückgängig gemacht (das Element sieht wie vor der Animation aus)

  • forwards ... das Element sieht so aus, wie im letzten Keyframe definiert. ACHTUNG: der "letzte" Keyframe hängt auch von animation-direction und animation-count ab.

  • backwards ... das Element sieht so aus, wie im ersten Keyframe definiert. ACHTUNG: der "erste" Keyframe hängt von der animation-direction ab. Wenn ein animation-delay definiert ist, wird der erste Keyframe sofort angewandt, d. h. während des delay sieht das Element schon so aus, wie im ersten Keyframe definiert.

  • both ... "sowohl als auch": wenn ein animation-delay definiert ist, wird das Element sofort wie im ersten Keyframe definiert, dargestellt. Und nach der Animation, bleibt es wie im letzten Keyframe definiert.

Abschnitt 31: Modern CSS

272. Variables and Media Queries

CSS Variables werden normalerweise in :root definiert (eine Pseudoklasse, die das "Haupt"element repräsentiert - im Normalfall html). Außerdem hat :root eine SEHR hohe Spezifität, d. h. es gilt "immer".


Man kann dann den Wert einer CSS Variablen in Abhängigkeit einer Media Query ändern (um beispielsweise margin oder padding an die Größe des Viewports anzupassen). Das geschieht so: 


:root {
  --margin-large: 2rem;
}
p {
  margin: var(--margin-large);
}
@media only screen and (max-width: 800px) {
  :root {
    --margin-large: 1rem;
  }
}

Wo immer man dann var(--padding-large) verwendet wird 2rem eingesetzt (sofern, die width vom Viewport > 800px ist, ansonsten 1 px). Man muss NICHT die einzelnen CSS-Anweisungen, die diese Variablen verwenden im @media Block wiederholen - diese gelten auch so mit dem geänderten Wert der Variablen. Im Beispiel hat jeder p ein margin von 2rem (>800px), sonst 1rem.

Damit kann man sich das Designen von responsive Webseite erleichtern.

275. The Rules of Calc()

calc() ... Unterstützt vier Operatoren: +, -, * und /. Man sollte vor und nach dem Operator immer ein Leerzeichen schreiben (auch wenn es technisch nur für + und - notwendig ist).

Man kann auch mehrere Operationen hintereinander ausführen (es gilt Punkt vor Strichrechnung).

Und man kann in der Berechnung auch auf CSS Variablen zugreifen:

width: calc(100% - 2 * var(--spacing));


Damit wird width auf 100% (vom Parent) gesetzt, es wird aber das zweifache von der --spacing Variablen abgezogen (beispielsweise, weil das Parent Element ein padding auf der Seite von jeweils einem var(--spacing) definiert hat.
